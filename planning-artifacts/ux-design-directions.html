Prevention Rule
Never run prisma migrate dev in CI/CD environments. Reserve it for local development only. Use prisma migrate deploy in all automated pipelines.
Fix
Replace migrate dev with migrate deploy in your CI workflow file. The deploy command applies pending migrations without resetting the shadow database or generating new migrations.
# .github/workflows/deploy.yml - npx prisma migrate dev # ❌ resets shadow DB + npx prisma migrate deploy # ✅ safe for CI
